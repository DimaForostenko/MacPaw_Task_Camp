Створення цього API вимагає кількох етапів, включаючи налаштування бази даних, розробку серверної частини на PHP, створення RESTful ендпоінтів та додавання необов'язкової функціональності, такої як авторизація. Ось загальний план для створення цього додатку:

### Етап 1: Налаштування бази даних

1. Створіть базу даних та таблиці згідно вказаних вимог.
2. Встановіть зв'язок з базою даних у вашому PHP-додатку, наприклад, використовуючи PDO або Eloquent (для Laravel).
3. Розробіть міграції для створення таблиць у використовуваному фреймворку.

### Етап 2: Створення API ендпоінтів

1. Розробіть контролер або роутер для обробки HTTP-запитів.
2. Реалізуйте POST-запит для створення нових зборів (створення запису в таблиці "collections").
3. Реалізуйте GET-запит для отримання списку всіх зборів (запит до таблиці "collections").
4. Реалізуйте POST-запит для додавання внесків (створення запису в таблиці "contributors").
5. Реалізуйте GET-запит для отримання деталей конкретного збору за його ідентифікатором (запит до таблиць "collections" і "contributors").
6. Реалізуйте GET-запит для фільтрації зборів за залишеною сумою (запит до таблиці "collections" з відповідним умовним оператором).

### Етап 3: Додаткові можливості (за бажанням)

1. Реалізуйте PUT- та DELETE-запити для редагування та видалення зборів та внесків.
2. Додайте механізм авторизації через токени для забезпечення доступу до API.

### Етап 4: Валідація та обробка помилок

1. Додайте валідацію даних для запитів, щоб переконатися, що вони відповідають вимогам.
2. Реалізуйте обробку помилок і повертайте відповідні HTTP-статуси та повідомлення у відповіді.

### Етап 5: Документація та коментарі

1. Додайте коментарі до вашого коду, щоб пояснити, які функції виконуються.
2. Створіть документацію до вашого API, описавши доступні ендпоінти, їх параметри та повернуті дані.

Це загальний план для створення запропонованого API. Важливо пам'ятати про забезпечення безпеки, оптимізації та тестування вашого додатку перед випуском в продакшн.
